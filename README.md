## ADODEVMETRICS

Инструмент для сбора статистики коммитов и pull request'ов из Azure DevOps репозитория с выгрузкой в Excel.

Подключается к Azure DevOps REST API, выбирает все коммиты из заданной ветки за последние N месяцев, собирает метрики по каждому коммиту и PR, группирует по авторам и сохраняет детальный отчёт в Excel-файл.

## Возможности

- Выборка коммитов из указанной ветки за последние N месяцев
- Подсчёт только тех коммитов, где `author` и `committer` совпадают
- По каждому коммиту: автор (email), дата, hash, сообщение, строк добавлено / удалено
- Количество коммитов на автора
- Статистика по pull request'ам: создано, одобрено, прокомментировано
- Детализированный Excel-отчёт (строка = коммит)
- Имя файла генерируется автоматически: `{repo}_{YYYYMMDD_HHMMSS}.xlsx`

## Технологии

- Python 3.8+
- [azure-devops](https://github.com/microsoft/azure-devops-python-api) — SDK для Azure DevOps REST API
- [openpyxl](https://openpyxl.readthedocs.io/) — генерация Excel-файлов
- [python-dotenv](https://github.com/theskumar/python-dotenv) — загрузка конфигурации из `.env`

## Настройка

Создайте файл `.env` в корне проекта:

```env
ADO_ORGANIZATION_URL=https://dev.azure.com/your-org
ADO_PROJECT=MyProject
ADO_REPOSITORY=MyRepo
ADO_BRANCH=main
ADO_PAT=your-personal-access-token
MONTHS_BACK=3
```

Необходимые права PAT-токена:
- `Code` — Read
- `Pull Request Threads` — Read

## Запуск

```bash
pip install -r requirements.txt
python main.py
```

## Формат Excel-отчёта

Каждая строка — один коммит:

| Автор (email) | Дата | Hash | Сообщение | Добавлено строк | Удалено строк |
|---|---|---|---|---|---|

Дополнительно для каждого автора собирается сводка по PR: создано, одобрено, прокомментировано.

## Ограничения и оптимизация

Инструмент рассчитан на работу с **высоконагруженными репозиториями** (~3M строк кода, ~100 PR/неделю):

- **Пагинация** — обход коммитов и PR постранично (`$top` / continuation token), без загрузки всего списка в память
- **Батчинг** — запросы группируются пакетами, а не выполняются поодиночке
- **Rate limiting** — автоматический retry с exponential backoff при получении `429 Too Many Requests`
- **Серверная фильтрация** — диапазон дат и ветка передаются параметрами запроса, фильтрация не выполняется на клиенте
- **Прогресс** — вывод текущего прогресса в консоль при обработке больших объёмов данных
